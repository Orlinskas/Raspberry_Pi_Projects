# Рекомендации по покупке набора для робота robot_prome_v1

[English](BUYING_GUIDE.md)

Краткий чек-лист: что купить, чтобы собрать робота под этот проект.

---

## Что нужно проекту (из кода)

| Компонент | Назначение в проекте |
|-----------|----------------------|
| **Raspberry Pi** | Мозг: Python, OpenCV, Ollama (или облачная LLM), GPIO |
| **Драйвер моторов L298N** | Два канала PWM (IN1–IN4, ENA, ENB) — езда вперёд/назад и повороты |
| **2 DC-мотора** | Колёса: шаги вперёд/назад, повороты на месте |
| **Ультразвуковой датчик HC-SR04** | `obstacle_cm` — дистанция до препятствия впереди |
| **Камера** | OpenCV, 640×480: картинка для LLM и видеопоток в браузере |
| **RGB-светодиод** | LIGHT_ON / LIGHT_OFF, индикация (опционально — можно заглушить в коде) |
| **Питание** | Батарейки/аккумулятор на 6–12 В для моторов + 5 В для Pi |

В `settings.py` зашиты пины под эту связку: L298N (IN1–IN4, ENA, ENB), HC-SR04 (TRIG/ECHO), RGB (3 пина), при необходимости — серво на одном пине.

---

## Варианты покупки

### 1. Готовый набор «машинка на Raspberry Pi» (самый быстрый старт)

Ищите наборы вроде:

- **«Raspberry Pi Robot Car Kit»** / **«2WD Smart Car Kit»** с Raspberry Pi  
- В комплекте обычно: шасси с 2 колёсами, L298N (или аналог), 2 DC-мотора, HC-SR04, иногда шасси с креплением под Pi и камеру.

**Проверьте в описании:**

- Совместимость с **Raspberry Pi** (GPIO, 5 V).
- Наличие **L298N** (или совместимого драйвера с IN1–IN4 и ENA, ENB).
- **Ультразвуковой датчик** (HC-SR04 или совместимый).
- Место под **камеру** (USB-вебкамера или CSI) и при желании — под батарейный блок.

Если в наборе другой драйвер (например, на TB6612) — пины в `settings.py` нужно будет переназначить под вашу распайку.

---

### 2. Собрать самому из отдельных модулей

| Что купить | Примеры | Замечание |
|------------|---------|-----------|
| Raspberry Pi 4 (2–4 GB) или Pi 5 | Официальные или партнёрские | Pi 3 тоже подойдёт, если LLM в облаке. |
| Драйвер L298N | Двухканальный модуль | Часто идёт с радиатором. |
| Шасси 2WD | Два колёса + моторы, платформа | Под размер Pi и батареи. |
| HC-SR04 | Ультразвуковой датчик | TRIG + ECHO, питание 5 V. |
| USB-камера | Любая с Linux/OpenCV | Либо Raspberry Pi Camera (CSI). |
| RGB LED (опционально) | 1 шт. общий катод/анод | Или не подключать и не использовать LIGHT_* в коде. |
| Батарея | 2S LiPo 7.4 V или 6×AA-держатель | Отдельно 5 V для Pi (Step-down или PowerBank). |
| Провода, макетная плата | Dupont, breadboard | Для подключения GPIO. |

Пины в проекте (BCM): моторы IN1=20, IN2=21, IN3=19, IN4=26, ENA=16, ENB=13; датчик ECHO=0, TRIG=1; RGB: 22, 27, 24. При другой распайке меняйте константы в `settings.py`.

---

### 3. Краткие советы

- Готовый набор: убедитесь, что в нём **L298N** и **HC-SR04** и что он заявлен под **Raspberry Pi** — тогда меньше возни с перепайкой.
- Камера: любая USB с поддержкой в Linux или официальная Pi Camera; в коде используется индекс 0 и 640×480.
- Мозг робота — облачная LLM через Ollama; мощный Pi не обязателен для логики, но нужен для запуска Python, камеры и GPIO.
- Все пины настраиваются в одном файле `settings.py` — под свой набор можно поменять только там.

## Минимальный набор для «сухого» запуска (без железа)

Чтобы только проверить логику и камеру на своём компе:

- Python 3.8+, OpenCV, Ollama (или облачный эндпоинт).
- Запуск: `python main.py --mode dry` — моторы и датчик не используются.

Для полноценного робота нужны Raspberry Pi и перечисленные выше компоненты.
